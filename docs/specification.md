# 上限仕入れ額計算機 仕様書

作成日: 2026-06-15

## 1. 目的

想定売値から上限仕入れ額を即時計算し、その結果を起点に通常の電卓操作も行える軽量アプリを作る。

主用途は PC だが、スマホでも同じ機能を使えるレスポンシブ UI とする。計算結果はワンクリックまたはワンタップでコピーできる。

## 2. 推奨技術スタック

### 2.1 推奨構成

| 項目 | 採用候補 | 理由 |
| --- | --- | --- |
| UI | Vanilla TypeScript + HTML + CSS | 機能規模が小さいため、React などの大きな UI ランタイムを避けて軽量化する |
| ビルド | Vite | 開発サーバーと本番ビルドが速く、静的アプリとして配布しやすい |
| 計算 | decimal.js | 金額計算で JavaScript の浮動小数点誤差を避ける |
| 保存 | localStorage | 係数、履歴、表示設定だけなら十分。サーバー不要 |
| コピー | Clipboard API | Web 標準 API で結果コピーを実装できる |
| スマホ対応 | PWA | ブラウザから利用でき、ホーム画面追加とオフライン動作に対応しやすい |
| Windows アプリ化 | Tauri v2 を後続候補 | PC 配布が必要になった場合に、同じ Web UI を軽量なデスクトップアプリとして包める |

### 2.2 採用方針

初期版は **Vite + Vanilla TypeScript の PWA** として作る。

理由:

- 計算機アプリは画面数と状態が少なく、フレームワーク依存を増やすメリットが小さい。
- PWA にすると PC とスマホを同じコードで扱える。
- ネットワーク通信が不要なため、静的ファイルだけで動かせる。
- Windows のインストーラーが必要になった場合だけ Tauri 化すればよい。

Tauri は「主要デスクトップとモバイル向けの小さく速いバイナリを作る」用途に向くが、Rust 環境や配布設定が増えるため、初期 MVP では必須にしない。

### 2.3 調査メモ

- Vite は高速な開発サーバーと最適化された本番ビルドを提供する。公式ガイドでは Vanilla TypeScript テンプレートも用意されている。
- Tauri v2 は Windows/macOS/Linux に加え Android/iOS も対象にでき、システム WebView を使うため Electron より軽量にしやすい。
- PWA は manifest と service worker によってインストール可能・オフライン対応の Web アプリにできる。
- Clipboard API の `navigator.clipboard.writeText()` で計算結果をコピーできる。
- localStorage はブラウザ内に文字列キー/値を永続保存でき、係数や履歴の保存に適する。

- MDN Progressive Web Apps: https://developer.mozilla.org/en-US/docs/Web/Progressive_web_apps

## 3. 機能要件

### 3.1 上限仕入れ額計算

ユーザーが想定売値を入力すると、設定済み係数を掛けた上限仕入れ額を表示する。

計算式:

```text
上限仕入れ額 = 想定売値 * 係数
```

初期係数:

```text
0.65
```

例:

```text
想定売値: 10,000
係数: 0.65
上限仕入れ額: 6,500
```

### 3.2 係数設定

- デフォルト係数は `0.65`。
- 設定画面または設定パネルで変更できる。
- 入力可能範囲は `0` より大きく `1` 以下を基本とする。
- ただし業務上 `1` 超の係数を使う可能性を残す場合は、上限を `9.9999` 程度に拡張できる設計にする。
- 変更後は localStorage に保存する。
- リセット操作で `0.65` に戻せる。

### 3.3 電卓機能

上限仕入れ額の結果を起点に、任意の数値を使って追加計算できる。

必須演算:

- 加算 `+`
- 減算 `-`
- 乗算 `×`
- 除算 `÷`
- クリア `C`
- 直前入力削除 `⌫`
- イコール `=`

主な操作:

- 上限仕入れ額を表示したら、その値を電卓の現在値として読み込める。
- 例: `上限仕入れ額 6,500` から `- 500 = 6,000`。
- 例: `上限仕入れ額 6,500` から `/ 1.1 = 5,909.09...`。
- 計算結果をさらに次の計算へ継続利用できる。

### 3.4 履歴

Windows の計算機のように、過去の計算履歴を一覧表示できる。

履歴に残す対象:

- 想定売値から上限仕入れ額を算出した記録
- 電卓で `=` を押して確定した記録
- コピーした結果は履歴に「コピー済み」状態を付けてもよいが、必須ではない

履歴項目の例:

```text
売値 10,000 × 0.65 = 6,500
6,500 - 500 = 6,000
6,000 ÷ 1.1 = 5,454.5455
```

履歴仕様:

- 新しい履歴を上に表示する。
- 最大保存件数は初期値 `100` 件。
- localStorage に保存する。
- 履歴をクリックまたはタップすると、その結果を現在値へ再利用できる。
- 履歴削除、全履歴クリアを提供する。

### 3.5 コピー

コピー対象:

- 上限仕入れ額
- 電卓の現在結果
- 履歴内の各結果

動作:

- コピーアイコンを押すと数値のみをクリップボードにコピーする。
- コピー成功時は短いフィードバックを表示する。
- Clipboard API が利用できない環境では、対象値を選択しやすいフォールバック表示を行う。

### 3.6 表示形式

金額表示:

- 3 桁区切りカンマを表示する。
- 内部計算値と表示値は分離する。
- 小数は設定で扱えるようにする。

初期表示ルール:

- 上限仕入れ額は小数 0 桁、四捨五入。
- 電卓結果は小数がある場合のみ必要桁数を表示。
- 除算などで小数が長くなる場合は最大 8 桁程度で丸める。

設定候補:

- 小数桁数: `0` / `2` / `4` / `8`
- 丸め方式: 四捨五入 / 切り捨て / 切り上げ

## 4. 非機能要件

### 4.1 軽量性

- 初期ロード JS はできるだけ小さく保つ。
- React/Vue などは初期版では使わない。
- 画像アセットは最小限にする。
- 外部 API 通信は行わない。
- 計算、履歴、設定はすべてローカルで処理する。

### 4.2 対応環境

PC:

- Windows 10/11 の最新 Chrome / Edge
- 将来的に Tauri で `.exe` またはインストーラー配布

スマホ:

- iOS Safari
- Android Chrome

画面幅:

- PC: `1024px` 以上を主対象
- タブレット: `768px` 以上
- スマホ: `360px` 以上

### 4.3 オフライン動作

- PWA 化する場合、アプリ本体をキャッシュしてオフライン起動できる。
- 履歴と設定は端末ごとの保存とする。
- 複数端末同期は初期版では対象外。

### 4.4 セキュリティ・プライバシー

- 入力値、履歴、設定は外部送信しない。
- localStorage に保存するため、共有 PC での利用時は履歴クリア導線を明確にする。
- クリップボード操作はユーザー操作時のみ実行する。

## 7. 計算仕様

### 7.1 数値入力

- 入力は半角数字、カンマ、小数点を許可する。
- 内部処理前にカンマを除去する。
- 空文字は未入力として扱う。
- 負の売値はエラーにする。
- 係数は `0` 以下をエラーにする。

### 7.2 上限仕入れ額

```ts
limitPrice = Decimal(salePrice).mul(Decimal(coefficient))
```

丸め:

- `half-up`: 指定小数桁で四捨五入
- `floor`: 指定小数桁で切り捨て
- `ceil`: 指定小数桁で切り上げ

### 7.3 電卓

- 四則演算は decimal.js で行う。
- `0` 除算はエラー表示にする。
- エラー表示後に数字入力を始めたら新規入力として復帰する。
- `=` 確定時に履歴へ追加する。

## 10. テスト方針

単体テスト:

- 売値 `10000`、係数 `0.65` で `6500`
- 係数変更後の再計算
- カンマ付き入力 `10,000`
- 小数の丸め
- `0` 除算エラー
- 履歴最大件数の制限

E2E または手動確認:

- PC 幅で履歴が右側に表示される
- スマホ幅で縦積みになる
- 上限仕入れ額をコピーできる
- 履歴クリックで結果を再利用できる
- リロード後も係数と履歴が残る

## 11. 決定事項

- 初期係数は `0.65`。
- 初期版はローカル完結の PWA とする。
- PC を主対象にしつつ、スマホでも同じ機能を使えるようにする。
- 軽量性を優先し、初期版では大きな UI フレームワークを使わない。
